Notitia

It was a group project in which myself, and other two team members, made a social platform in which people can meet up and exchange their knowledge on a certain subject with other people. I was responsible for implementing Google Maps API and adjusting it so it would improve user experience and ease of locating other users willing to exchange their services.For example, while filling out the form the user is asked to pin point his desired location by either selecting it on a map, or by putting in the desired address that was automatically stored in a database, and later shown on a map. Book-finder

Our second project required us to create a RESTful up with at least one API hooked up, and a login and register form for the user. I focused on creating a database for book exchange points where people could exchange books freely between each other. The two APIs I used were Google Books and Google Maps. Their main purpose was to improve user experience and help locate where a given book may be, by giving a search option for to book that the user wants to upload, and by giving it's location on the map.

HTML5, SASS, JavaScript ES6, JQuery 3.10, Gulp, Yarn, Git, GitHub, MongoDB, Mongoose, Express, Amazon Web Services (S3 Bucket).

Reversi Game

That was a first project we had to do at General Assembly that we were asked to do after only two weeks of learning how to code. The premise was simple. We come up with a game, then we code in Javascript. I decided to make Othello/Reversi board game in which each player has to flank enemies playing pieces in order to have fill out the board with as many pieces of his own.
One of the biggest challenges was to create a system of coordinates that would allow implementing a game logic would be consistent with rules of the game. The solution to this problem was to calculate the distance between each piece by taking into account the index number of each square, and applying a set of rules to the code that would allow only legal move.

JavaScript, jQuery, HTML, CSS and SASS.
